## Root Cause Identified **This is a Claude Code bug, not our implementation.** ### Known Claude Code Issues 1. **Issue #10373** - SessionStart hooks not working for new conversations - additionalContext is NOT injected for brand new conversations - Works after /clear, /compact, or URL resume - But fails silently on fresh session start 2. **Issue #13650** - SessionStart hook stdout silently dropped - Valid JSON with exit code 0, but output is dropped 3. **Issue #14281** - additionalContext injected multiple times (different bug) ## Workaround Implemented Created SDK session warning system (`session_warning.py`) that: 1. Shows orchestrator instructions on SDK initialization 2. Prints visible banner to stderr (which Claude sees) 3. Requires explicit dismissal: `sdk.dismiss_session_warning()` 4. Persists dismissal state in `.htmlgraph/.session-warning-state.json` This ensures agents always see orchestrator instructions regardless of Claude Code bug. ### Usage ```python from htmlgraph import SDK sdk = SDK(agent="claude") # Warning automatically shown if not dismissed for this session # Agent dismisses after reading (as first action) sdk.dismiss_session_warning() ``` ## Status - Workaround implemented and tested - Waiting for Claude Code fix to issue #10373 - Will remove workaround when upstream fix is available
