Implementation Summary

Added transition time analytics to track time spent in transitional activities (session-init, transition, conversation-init spikes) versus regular feature work.

New Analytics Method:

sdk.analytics.transition_time_metrics(session_id=None, start_date=None, end_date=None)

Returns dictionary with:

  • transition_minutes: Time in transition spikes
  • feature_minutes: Time in regular features
  • total_minutes: Combined time
  • transition_percent: % time in transitions (0-100)

Usage in CLI:

# Single session analysis
htmlgraph analytics --session-id session-123

# Project-wide analytics
htmlgraph analytics

# Recent sessions
htmlgraph analytics --recent 5

Interpretation:

  • <15%: Low transition time - Focused work
  • 15-30%: Moderate transition time - Normal
  • >30%: High transition time - Lots of context switching
