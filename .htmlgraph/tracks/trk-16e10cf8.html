Orchestrator Pattern: Explorer & Coder Subagents

Planned High Priority Created: 2025-12-26

Implement orchestration pattern where main agent spawns specialized subagents: - Explorer subagents: Codebase discovery, analysis, context gathering - Coder subagents: Implementation, testing, file modifications Benefits: - Main session context reserved for orchestration decisions - Parallel exploration and coding - Better context efficiency (subagents are ephemeral) - More tasks completed before context fills up

Overview

Create SubagentOrchestrator framework for spawning explorer and coder subagents

Context

Main agents fill context with exploration/coding. Delegating to subagents preserves orchestration capacity.

Acceptance Criteria

  1. ⏳ sdk.spawn_explorer() spawns explorer subagent via Task tool
  2. ⏳ sdk.spawn_coder() spawns coder subagent via Task tool
  3. ⏳ Subagent results automatically update HtmlGraph
  4. ⏳ Main session can orchestrate 3+ parallel subagents
  5. ⏳ Context usage reduced by 50%+ for exploration tasks
