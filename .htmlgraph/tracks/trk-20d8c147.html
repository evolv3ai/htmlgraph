Auto-Generated Transition Spikes for Session Management

Planned High Priority Created: 2025-12-25

Automatically create spikes for transitional periods to provide clear attribution boundaries

Overview

Implement auto-spike generation for session transitions to ensure complete activity attribution

Context

Current session management attributes ALL activities to features, causing: - Transitional activities (session start, feature switching) marked as drift - Loss of visibility into planning/review time - Incorrect attribution of pre/post-feature work

Requirements

⏳ Extend Node model with spike_subtype and auto_generated fields

must-have

⏳ Auto-create session-init spike at session start

must-have

⏳ Auto-create transition spike when feature completes

must-have

⏳ Update attribution logic to prioritize active spikes

must-have

⏳ Add dashboard visualization for transition spikes

should-have

⏳ Add analytics for transition time metrics

must-have

Acceptance Criteria

  1. ⏳ All activities attributed to feature OR spike No orphaned/drifted activities
  2. ⏳ Transition spikes auto-created and completed Timeline shows clear boundaries
  3. ⏳ Analytics shows time spent in transitions vs features
