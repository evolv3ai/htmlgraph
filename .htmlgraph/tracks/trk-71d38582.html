TrackSpecPlan

Graph Database Completeness

In Progress High Priority Created: 2025-12-22

Expand HTML querying beyond CSS selectors and implement missing graph database properties

Overview

HtmlGraph needs to fully embody graph database characteristics while maintaining the "HTML is All You Need" philosophy. This track addresses two key gaps: (1) Limited query capabilities restricted to CSS selectors, and (2) Missing graph database properties like bi-directional edges and efficient traversal.

Context

Current limitations: CSS selectors cannot express OR/NOT logic, numeric comparisons, text search, or multi-hop traversals. Edges are uni-directional requiring O(V×E) reverse lookups. No query composition API exists.

Acceptance Criteria

  1. ✅ Reverse edge lookup completes in O(1) time EdgeIndex with dict-based lookups
  2. ✅ Query builder supports method chaining for complex queries sdk.query().where().and_().or_()
  3. ✅ find() method matches BeautifulSoup API patterns find(type="feature", status="blocked")
  4. ✅ All query types have >95% test coverage 134 tests for Phases 1-4
  5. ⏳ Documentation includes migration guide from CSS-only queries docs/migration.md exists
  6. ⏳ Performance benchmarks show <100ms for 10K node graphs benchmark suite passes
  7. ⏳ JS library has feature parity with Python SDK feature matrix comparison
  8. ✅ Existing CSS selector queries remain backward compatible all existing tests pass
