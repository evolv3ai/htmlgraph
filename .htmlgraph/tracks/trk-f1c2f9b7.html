TrackSpecPlan

Orchestrator Enforcement System

Planned High Priority Created: 2025-12-30

Overview

Implement a robust system to enforce orchestrator delegation patterns in Claude Code. **Problem**: Claude executes implementation work directly instead of delegating to subagents, even when orchestrator skill is activated. **Solution**: 5-component system with smart PreToolUse hook, mode state management, CLI commands, and clear enforcement rules. **Success Criteria**: - Claude consistently delegates implementation work in orchestrator mode - Single Read/Grep lookups still work (not blocked) - Work item creation via SDK unblocked - Clear error messages with Task suggestions when blocking - Users can easily disable mode when needed - No false positives blocking legitimate operations - All tests pass with >90% coverage

Requirements

⏳ Block direct implementation (Edit/Write) when orchestrator mode is active

must-have

⏳ Allow lightweight orchestrator operations (single lookups, SDK calls, Task delegation)

must-have

⏳ Provide clear Task tool suggestions when blocking operations

must-have

⏳ Support strict (blocking) and guidance (warning) enforcement levels

must-have

⏳ CLI commands to enable/disable/check orchestrator mode

must-have

⏳ Auto-activate when features exist, auto-deactivate when none

must-have

⏳ Comprehensive test coverage for all enforcement rules

must-have

⏳ Clear documentation and error messages

must-have
